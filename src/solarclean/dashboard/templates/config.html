Site location and timezone

Click the map (or type coordinates) and press "Apply to config above". The location's IANA timezone is detected automatically from the coordinates. This updates site.latitude, site.longitude, both configured timezone fields, and the simulation start/end UTC offsets in the editor — including daylight-saving offsets for their local dates. Then validate and save as usual. The Default configuration uses the nasa_power weather provider, so the chosen location changes the real weather the simulation runs on: hourly irradiance (GHI/DNI/DHI), air temperature, wind speed, relative humidity, and precipitation are fetched from NASA POWER for those coordinates (once per location and period, then cached under data/cache/weather). Rainfall totals, seasonal soiling, event-tape dates, and cleaning decisions are grouped by the selected local timezone. Dust-event frequency and cost calibration stay Riyadh central-v2 regardless of location, but humidity-coupled soiling (dew cementation) and coating condensation follow the fetched weather when enabled in the config.
