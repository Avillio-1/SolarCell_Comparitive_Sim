Site location (latitude / longitude)

Click the map (or type coordinates) and press "Apply to config above" to update site.latitude / site.longitude in the editor — then validate and save as usual. The Default configuration uses the nasa_power weather provider, so the chosen location changes the real weather the simulation runs on: hourly irradiance (GHI/DNI/DHI), air temperature, wind speed, relative humidity, and precipitation are fetched from NASA POWER for those coordinates (once per location and period, then cached under data/cache/weather). Two things do not follow the map: soiling/dust-event and cost calibration parameters remain the Riyadh central-v2 assumption set wherever the site is placed, and with the fixture or csv providers the weather data is fixed, so coordinates are recorded as metadata only.
